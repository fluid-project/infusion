Renderer Example: Selector-based Rendering

The table below is, initially, the template that will be used by the Fluid Renderer. The elements in the mark-up do not have an "rsf:id" attribute to identify them to the renderer. Instead, selectors (in this example, class-based) are used to identify the elements.

Each of the "Render..." buttons will render the table using a different version of the component tree.

NOTE: Each rendering internally starts with a clean version of the template.
